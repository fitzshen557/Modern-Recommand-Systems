> **论文**：Deep Neural Networks for YouTube Recommendations  
> **来源**：Google / YouTube  
> **发表**：RecSys 2016  
> **论文链接**：https://dl.acm.org/doi/10.1145/2959100.2959190

---

## 一、为什么这篇论文是工业推荐的圣经

YoutubeDNN 是第一篇把深度学习推荐系统从"研究 demo"推向"真实亿级用户生产系统"的论文。

它有以下几个"第一"：
1. **第一次系统性描述工业推荐的双阶段架构**（召回 + 精排）
2. **第一次把用户历史行为序列作为召回模型的核心特征**
3. **第一次在工业规模验证** In-batch Negative Sampling 的有效性
4. 精排部分的**加权逻辑回归处理观看时长**，至今仍是行业标准

---

## 二、整体架构：召回 + 精排两阶段

```
YouTube 视频库（数十亿）
        ↓
   【召回模型】
   从数十亿 → 数百候选
        ↓
   【精排模型】
   从数百 → Top-K 最终列表
```

召回关注**覆盖率**，精排关注**精准度**。两个阶段解耦，是至今工业界的标准范式。

## 三、召回模型详解

### 3.1 把召回建模成分类问题

YouTube 的天才设计：**把"推荐哪个视频"建模成一个极大规模的多分类问题**。

用户 $u$ 在时刻 $t$ 的上下文 $C$ 下，视频 $i$ 被推荐的概率：
$$P(\text{watch} = i \mid u, C) = \frac{e^{v_i \cdot u}}{\sum_{j \in V} e^{v_j \cdot u}}$$

- $u \in \mathbb{R}^d$：用户向量（由神经网络生成）
- $v_i \in \mathbb{R}^d$：视频 $i$ 的向量（Embedding 表中直接查）
- $V$：全量视频库

这本质上是一个 Softmax 分类器，类别数 = 视频数量（数十亿）。

### 3.2 用户侧特征工程

用户塔的输入特征（这部分设计极为精妙）：

| 特征 | 类型 | 处理方式 |
|------|------|---------|
| 观看历史（视频 ID 序列） | 稀疏序列 | Embedding → 平均池化 |
| 搜索历史（query token 序列） | 稀疏序列 | Embedding → 平均池化 |
| 人口统计（性别/年龄/地区） | 稠密/稀疏 | 直接输入或 Embedding |
| Example Age（样本年龄）| 稠密 | 关键特征，见下 |

**Example Age 的妙处**：训练样本有时效性偏差——越老的视频训练样本越多，模型倾向推老内容。Youtube 用"样本生成时距当前的天数（取反）"作为特征，让模型显式感知时效性。推断时固定设为 0（表示"新内容"），从而消除偏差。

### 3.3 负采样策略

全量 Softmax 在视频数十亿时不可计算。YouTube 用 **In-batch Negative Sampling**：

- **正样本**：用户实际观看的视频
- **负样本**：同一 batch 中其他用户观看的视频（当作负样本）
- 对热门视频做**频率降权**（热门视频天然作为负样本的概率高，需要校正）

$$\mathcal{L} = -\log \frac{e^{u \cdot v^+}}{e^{u \cdot v^+} + \sum_{j \in \text{batch}} e^{u \cdot v_j^-}}$$

### 3.4 推断：近似最近邻检索

训练完成后：
- 所有视频向量 $v_i$ 离线计算存入索引
- 在线：实时算用户向量 $u$，ANN 检索 Top-K 视频

当年 YouTube 用的是**Locality-Sensitive Hashing (LSH)**，今天已被 FAISS/ScaNN 替代。

## 四、精排模型详解

### 4.1 特征更丰富，模型更深

精排只处理召回的几百候选，可以用**更多特征和更复杂的模型**：

- 召回候选的视频特征（标题、频道、标签等）
- 用户与该视频的历史交互（是否看过同频道/同类视频）
- 上下文特征（时间、设备、网络类型）

### 4.2 优化观看时长，而非点击率

这是 YouTube 精排最重要的设计决策。

**问题**：如果直接优化 CTR（点击率），模型会推大量"标题党"——标题吸引人但内容差，用户点了马上关掉。

**解法**：用**加权逻辑回归**，正样本权重 = 实际观看时长（秒），负样本权重 = 1：

$$\text{expected watch time} \approx \frac{e^{Wx+b}}{1 + e^{Wx+b}} \times T_{\text{exposure}}$$

在推断时，用 $e^{Wx+b}$（Sigmoid 前的 logit 的指数）作为最终排序分，近似等于"期望观看时长"。

## 五、关键工程细节

| 细节 | 内容 |
|------|------|
| **Serving 架构** | 召回 + 精排完全分离，分别部署 |
| **异步训练** | 不做实时更新，离线批量训练，周期性更新模型 |
| **特征归一化** | 连续特征做归一化 + 平方根/对数变换 |
| **训练数据生成** | 直接用用户日志，不做特殊过滤（包含各种噪声，但规模大） |
| **负采样纠偏** | 热门视频在 In-batch 中出现频率高，需要对打分做频率校正 |

## 六、YoutubeDNN 后续的演进

| 问题 | 改进方向 | 代表工作 |
|------|----------|----------|
| 单向量无法表达多兴趣 | 多兴趣向量 | MIND (2019) |
| 行为序列简单平均 | 注意力加权 | DIN (2018) |
| 负采样不够 hard | 难负样本挖掘 | DNS (2021) |
| 曝光偏差 | 去偏学习 | DSSM+IPS |
| 冷启动物品无向量 | 内容特征/LLM | MoRec (2023) |

## 七、懂哥点评

YoutubeDNN 是推荐系统工程师的必读论文，没有之一。它的价值不在于模型有多复杂（其实很简单），而在于它**把大量工业直觉系统化表达了出来**：

1. **召回/精排分离**：这不是技术决定，是延迟和精度 trade-off 的工程决定
2. **Example Age**：处理时序偏差的神来之笔，一个特征解决一个根本性问题
3. **优化观看时长而非 CTR**：目标函数的选择比模型结构更重要

读这篇论文时重点不要放在模型结构上（很普通），要关注**每个设计决策背后的工程逻辑**。
