# READER — 复购感知的 GMV 延迟反馈建模

> 论文：READER: Recurrence-Enhanced Multi-task Learning for Post-Click GMV Prediction with Delayed Feedback
> 机构：阿里妈妈
> 会议：WWW 2026
> arXiv：2601.20307

---

## 一、背景与动机

### 从 CVR 到 GMV：后链路预估的升级

CVR 预估只回答"用户会不会转化"，但电商广告真正关心的是 **GMV（成交金额）**——转化了还要看买了多少钱。

```
传统链路：CTR → CVR → 排序
升级链路：CTR → CVR → GMV → 排序
                          ↑
                    READER 解决的问题
```

### 两大核心难题

**难题一：延迟反馈**

GMV 的延迟比 CVR 更严重：
```
CVR 延迟：点击 → 下单（几分钟~几天）
GMV 延迟：点击 → 下单 → 收货 → 确认 → 退款结算（几天~几周）
```
训练时能观测到的 GMV 只是"部分已完成"的，大量复购 GMV 还没发生就被截断了。

**难题二：复购行为的异质性**

用户对同一商品的消费模式差异极大：
```
单次购买用户：买一次就走，GMV = 单次订单金额
复购用户：    反复购买，GMV = 多次订单累积金额（可能是单次的 5~10 倍）
```

一个模型统一建模这两类用户，会被大量"单次购买"用户拉偏，低估复购用户的真实 GMV。

## 二、方法详解

### 整体框架

```
输入特征 x
    │
    ▼
[共享底层 Embedding]
    │
    ├──► [Router 路由模块] → 判断是否为复购用户
    │         │
    │    单次购买 ──► [Single-Purchase Expert] → GMV₁
    │    复购用户 ──► [Recurrence Expert]       → GMV_recur
    │
    └──► 动态加权融合 → 最终 GMV 预估
```

### 核心模块一：复购感知路由（Recurrence-Aware Router）

Router 先预估"该用户对该商品的复购概率"：

```
P(复购) = sigmoid(Router(x))

Router 输入特征：
  - 用户历史复购行为（同类目/同商品复购次数）
  - 商品复购率（该商品的平台平均复购率）
  - 用户-商品交互频率
  - 会员状态、消费层级
```

根据复购概率动态混合两个 Expert 的输出：

```
GMV_pred = P(复购) × GMV_recur + (1 - P(复购)) × GMV₁
```

### 核心模块二：复购强度建模（Recurrence Expert）

复购用户的 GMV 不是一次性的，而是时序累积的。READER 用**循环结构（GRU）**建模复购序列：

```
历史购买序列：[order₁, order₂, ..., orderₜ]
    │
    ▼
GRU 编码 → 复购强度表示 h_recur
    │
    ▼
预测未来 N 次购买的期望 GMV
```

### 核心模块三：延迟反馈的动态目标校准

训练时 GMV 标签是"截止当前已观测到的累积 GMV"，存在欠估计。READER 用**动态目标校准（Dynamic Target Calibration，DTC）**修正：

```
GMV_observed(t) = GMV_真实 × 完成率(t)

完成率(t) 估计：
  - 根据历史数据，统计不同类目商品在 t 天后的"GMV 完成比例"
  - 用这个比例对当前截断的 GMV 做放大校正

校正后标签：
  GMV_calibrated = GMV_observed(t) / 完成率(t)
```

### 核心贡献：TRACE 数据集

READER 论文同时发布了首个 **GMV 延迟反馈公开基准数据集 TRACE**：

```
TRACE 数据集特点：
  - 真实电商广告日志
  - 包含完整的时序 GMV 标签（不只是截断观测）
  - 覆盖不同类目（快消 / 服装 / 3C 等）延迟分布
  - 是 GMV 延迟反馈研究的第一个标准 benchmark
```

## 三、实验结果

### 离线实验（TRACE 数据集）

| 模型 | GAUC | GRMSE | 说明 |
|------|------|-------|------|
| 直接回归 GMV | baseline | baseline | 忽略延迟反馈 |
| ESMM 风格联合建模 | +0.8% | -3.2% | 全空间建模但无复购区分 |
| DFM（参数化延迟） | +1.1% | -4.5% | 延迟建模但无复购区分 |
| **READER** | **+3.4%** | **-11.2%** | 复购路由 + 动态校准 |

### 线上 A/B 实验

```
指标               提升幅度
GMV（成交金额）     +5.2%
ADVV（广告主价值）  +3.8%
复购用户 GMV       +12.6%   ← 复购用户收益尤其显著
单次购买用户 GMV    +1.2%   ← 单次用户也有小幅提升
```

## 四、懂哥点评

### 核心价值

**READER 最聪明的地方是"把用户分层"这一步。**

过去 GMV 预估都把所有用户混在一起建模，但单次购买用户和复购用户的 GMV 生成机制完全不同——前者是一次性事件，后者是时序过程。用一个模型硬学，会导致复购用户的 GMV 被严重低估（因为单次购买用户数量更多，权重更大）。

路由机制让模型"先判断你是哪类用户，再用对应的子模型预测"，这和医学上"先诊断再治疗"的逻辑一样，物理意义清晰。

### 工业落地价值

```
最适合的场景：
  ✅ 高复购率类目（快消/食品/日用品）
  ✅ 平台有会员体系（会员复购率明显高于非会员）
  ✅ 广告主按 GMV 出价（oCPM/ROI 出价）

收益有限的场景：
  ❌ 低复购率类目（3C/家电，基本一次性购买）
  ❌ 纯 CPC 出价（不关心 GMV，只关心点击）
```

### 局限性

1. **Router 的冷启动**：新用户没有历史复购记录，路由判断依赖冷启动处理
2. **动态校准依赖历史分布**：如果类目的延迟分布发生变化（如大促期间），完成率估计会偏
3. **GRU 复购建模的序列长度**：长期复购用户序列很长，工程上需要截断处理
4. **TRACE 数据集局限**：只覆盖阿里妈妈的电商场景，其他平台需要重新验证

### 与 TESLA 的配合

```
TESLA 解决：CVR 延迟 + 退款率（净转化率）
READER 解决：点击后 GMV 延迟 + 复购异质性

两者互补，可以级联使用：
  TESLA → 预测 NetCVR（净转化概率）
  READER → 预测转化后的期望 GMV
  最终排序分：NetCVR × GMV_pred
```
