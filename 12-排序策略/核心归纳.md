## 🧠 核心问题框架

### Pointwise vs Pairwise vs Listwise

```
Pointwise：
  每个 item 独立打分，预测绝对值（CTR/CVR）
  优点：简单，工程成本低
  缺点：忽略 item 间竞争关系，局部最优≠列表最优

Pairwise：
  对 item 对建模，预测相对顺序
  代表：RankNet、LambdaRank
  优点：比 pointwise 更关注排序
  缺点：候选对数量 O(n²)，效率低

Listwise：
  直接对整个候选列表建模，优化列表级指标
  代表：ListNet、SetRank、GRPO-Rank
  优点：建模 item 间交互，全局最优
  缺点：计算复杂度高，在线部署挑战大
```

### 融合公式设计的三大矛盾

```
矛盾一：目标权重 vs 目标相互制约
  加法：权重直观，但各目标独立（某目标过高会主导）
  乘法：自动约束，但无法单独调节某目标权重

矛盾二：全局统一权重 vs 用户/场景差异化权重
  手工公式：所有用户用同一套系数
  自动搜参：可以学到场景级差异，但超参空间爆炸

矛盾三：工程简单性 vs 表达能力
  简单加权：上线快，可解释
  端到端 Listwise：效果好，但延迟/工程成本高
```
