# 混排（Blending & Fusion）

> 混排解决多路结果（推荐内容、广告、搜索插入、运营活动等）如何融合成**一个最优列表**的问题。
> 典型场景：信息流中推荐内容与广告的混合排列。

---

## 混排的核心问题

```
输入:
  推荐流: [内容A, 内容B, 内容C, ...]  （精排后）
  广告流: [广告X, 广告Y, ...]          （广告竞价后）
  运营流: [活动卡片, banner, ...]

输出: 一个统一的信息流列表，使得:
  max: 用户体验 (点击率/时长/满意度)
  s.t. 广告密度约束, 运营规则约束, 多样性约束
```

## 混排的难点

1. **目标异质**：推荐内容优化用户满意度，广告优化 RPM，两者目标不同
2. **约束复杂**：广告密度、最小间隔、频控等工程规则需要在排序中体现
3. **位置效应**：同一广告在第1位和第5位的点击率相差可达 3-5 倍，位置敏感性极强
4. **联合优化难**：推荐和广告分开优化是局部最优，联合优化需要统一价值单位
